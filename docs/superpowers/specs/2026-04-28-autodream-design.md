# Autodream 功能详细技术设计文档
## 一、设计概述
Autodream是Cognix的智能自进化核心模块，实现**记忆自动整理、习惯自动沉淀、技能自主进化**三大核心能力，参考借鉴了[NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent)的自我学习闭环设计理念，完全兼容现有Cognix记忆系统API，无破坏性变更。

### 核心设计目标
1. 零用户干预：所有整理、沉淀、进化过程在后台自动完成，不打扰用户正常使用
2. 全闭环自优化：从「数据采集→处理→沉淀→验证→优化」形成完整闭环，无需人工标注
3. 100%兼容性：完全复用现有记忆系统、事件总线、技能管理等核心模块，保持现有接口不变
4. 可插拔扩展：各模块独立解耦，支持自定义策略、算法替换

## 三、各模块详细设计

### 3.1 记忆自动整理流水线
#### 功能说明
自动处理所有记忆数据，保证记忆库的整洁、高价值、无冗余
#### 核心流程
1. **触发条件**
   - 定时触发：默认每日凌晨2点（低负载时段）
   - 事件触发：会话结束时、短期记忆达到阈值(50条)时
2. **处理步骤**
   | 步骤 | 实现逻辑 | 复用组件 |
   |------|----------|----------|
   | 去重 | 基于Jaccard相似度(阈值0.85)检测重复记忆，合并相同内容，保留最新版本 | `deduplicator.py` |
   | 冲突检测 | 语义比对识别矛盾记忆（如"喜欢用Python"和"禁用Python"），标记为待确认状态，不会自动覆盖 | `conflict_resolver.py` |
   | 压缩摘要 | 对短期记忆、历史会话进行LLM摘要提取，保留核心信息，丢弃低价值对话内容 | `memory_compressor.py` |
   | 自动分类 | 基于关键词+语义分类，自动写入user/settings/office/feedback/reference对应分类记忆库 | `memory_system.py` |
   | 归档清理 | 超过30天的低价值会话自动归档到sessions目录，超过180天的自动清理 | 新增逻辑 |
#### 核心接口
```python
def run_autodream_pipeline(trigger_type: str = "scheduled") -> Dict:
    """
    执行autodream记忆整理流水线
    :param trigger_type: 触发类型：scheduled/event_triggered/manual
    :return: 整理结果统计：新增记忆数、合并重复数、冲突数、归档数
    """
```

### 3.2 用户习惯自动识别引擎
#### 功能说明
从所有交互记录中自动挖掘沉淀用户习惯、偏好、行为模式，无需用户主动告知
#### 核心能力
1. **习惯分类**
   | 分类 | 识别内容 |
   |------|----------|
   | 办公习惯 | 周期性任务（每周周报/每月月报）、常用流程、联系人群组、审批路径、会议时间规律 |
   | 使用偏好 | 输出格式偏好（markdown/表格/脑图）、语言风格（正式/简洁/口语化）、工具选择偏好、通知方式偏好 |
   | 行为模式 | 高频操作、决策习惯、工作节奏（工作时段/响应速度）、常用快捷键/命令 |
   | 纠偏反馈 | 对AI输出的修正意见、禁止操作、要求遵守的规范 |
2. **识别逻辑**
   - 关键词匹配：优先匹配预设的习惯关键词库
   - 时序模式识别：基于至少3次重复出现的行为模式，判定为稳定习惯
   - 语义验证：调用LLM验证识别结果的准确性，置信度低于0.7的结果标记为待确认
#### 核心接口
```python
def extract_user_habits(time_range: Tuple[datetime, datetime] = None) -> List[Dict]:
    """
    提取指定时间范围内的用户习惯
    :param time_range: 时间范围，默认最近30天
    :return: 习惯列表，包含分类、内容、置信度、出现次数
    """
```

### 3.3 技能自我进化模块（参考Hermes Agent设计）
#### 设计参考（来自Hermes Agent）
- 自主技能创建：从复杂任务执行轨迹中自动生成可复用技能
- 技能自我改进：每次使用后自动优化技能逻辑、参数、错误处理
- 兼容开放标准：兼容[agentskills.io](https://agentskills.io)技能标准
- 技能版本管理：支持版本迭代、回滚、效果对比
#### 核心流程
```
任务执行成功 → 轨迹采集 → 模式识别 → 技能生成 → 效果验证 → 入库沉淀 → 使用优化
                                       ↑                          ↓
                                       └──────────────────────────┘
                                          每次使用后自动迭代
```
#### 核心能力
1. **技能自动生成**
   - 触发条件：同一类任务成功执行≥3次，或者单次复杂任务执行步骤≥5步且成功率100%
   - 生成逻辑：
     1. 提取任务的目标、输入参数、执行步骤、输出格式
     2. 自动生成标准化技能代码，符合Cognix技能开发规范
     3. 自动添加错误处理、参数校验逻辑
     4. 自动生成技能描述、使用示例
2. **技能自我优化**
   - 每次技能被调用后，自动收集执行日志、用户反馈
   - 识别执行失败、效果不佳的点，自动优化技能逻辑
   - 版本迭代：每次优化生成新版本，保留历史版本，支持回滚
3. **技能质量控制**
   - 自动生成测试用例，新技能必须通过测试才能上线
   - 调用成功率低于80%的技能自动下线，进入优化流程
   - 超过90天未被使用的技能自动归档
4. **标准兼容**
   - 技能存储格式完全兼容agentskills.io标准
   - 支持直接导入/导出社区公开技能
#### 核心接口
```python
def generate_skill_from_trajectory(trajectory_id: str) -> Optional[Skill]:
    """
    从任务执行轨迹生成技能
    :param trajectory_id: 执行轨迹ID
    :return: 生成的技能对象，生成失败返回None
    """

def optimize_skill(skill_id: str, execution_log: Dict) -> Skill:
    """
    基于执行日志优化技能
    :param skill_id: 技能ID
    :param execution_log: 执行日志
    :return: 优化后的新版本技能
    """
```

### 3.4 主动洞察反馈模块
#### 功能说明
基于记忆和习惯分析，主动给用户提供有价值的建议和提醒
#### 核心场景
1. 习惯优化建议：识别低效习惯，提供优化方案（如"你每周花2小时手动生成周报，可以使用自动生成周报技能"）
2. 遗漏事项提醒：基于周期性习惯，提醒即将到来的待办事项（如"明天是周一，记得准备周会材料"）
3. 最佳实践推荐：识别到当前任务可以使用已有技能完成，主动推荐
4. 风险预警：识别到操作可能和用户习惯冲突，提前提醒确认（如"你通常不会在周末修改生产环境代码，确认要继续吗？"）

### 3.5 自评估闭环模块
#### 功能说明
自动评估autodream所有产出的质量，持续优化算法效果，形成自学习闭环
#### 核心指标
| 指标 | 达标阈值 |
|------|----------|
| 记忆分类准确率 | ≥95% |
| 习惯识别准确率 | ≥90% |
| 技能调用成功率 | ≥85% |
| 主动建议采纳率 | ≥60% |
#### 优化逻辑
- 每月自动评估所有指标，低于阈值的模块自动触发算法调优
- 收集用户的纠正反馈作为负样本，训练优化分类、识别模型
- 定期清理低质量的记忆、习惯、技能数据
