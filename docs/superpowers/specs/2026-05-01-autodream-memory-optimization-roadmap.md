**状态**：草案  
**最后更新**：2026-05-01  
**核心设计原则**：纯Markdown可编辑优先、零依赖默认、组件可插拔、开关可控

---
## 一、总体设计原则（核心优势永远不变，基于现有成熟架构增量优化）
基于Cognix现有成熟的**双向存储设计理念**扩展，不重构核心架构：
1. **Markdown是唯一真相源**：所有原始记忆、偏好、规则100%存储在可直接编辑的Markdown文件中，用户修改Markdown等于直接修改记忆，不需要任何API/工具操作
2. **SQLite仅为可丢弃索引层**：所有结构化数据、索引、缓存、实体关系都是从Markdown自动提取的派生数据，可随时一键删除重建，不会丢失任何真实数据，损坏也不影响核心功能
3. **零依赖默认+组件可插拔**：默认版本依然零额外依赖开箱即用，所有企业级/高级能力都是可选项，按需开启
4. **开关可控+灰度升级**：所有新功能默认关闭，支持单独开启/关闭，出问题随时回滚，不影响现有业务
5. **数据100%可迁移**：纯文本格式永久兼容，仅复制Markdown文件夹即可完成备份/迁移/同步，不需要专属工具

## 三、实施保障原则
1. **开关可控**：所有新功能都有独立开关，默认关闭，可单独开启/关闭某类功能
2. **灰度上线**：新功能先小范围测试验证，没问题再全量上线
3. **数据可回滚**：所有新增的缓存、表、索引都可以一键删除重建，原始Markdown数据永远不会被修改
4. **兼容现有数据**：升级完全兼容现有记忆数据，不需要迁移，自动识别适配
