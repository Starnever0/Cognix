# Autodream 评估体系完整方案
## 一、设计概述
本评估体系完全适配Autodream的"Markdown+SQLite"记忆存储方案和五大核心模块能力，参考Mem0等行业最佳实践，实现**横向行业对比+模块专项测试+端到端效果验证**的三维度评估，所有测试可复现、可量化、贴合真实使用场景。

### 评估目标
1. 量化Autodream各模块的实际效果，验证功能正确性
2. 对比行业主流方案，明确Autodream的优势和待优化点
3. 建立基线版本，后续迭代可快速验证效果提升
4. 确保生产级性能和稳定性

---
## 二、整体评估框架
### 2.1 三个评估维度
| 维度 | 评估内容 | 目标 |
|------|----------|------|
| 🧪 **横向对比评估** | 和行业主流记忆方案做对比：<br>1. 基线方案：无记忆、全上下文填充、原生RAG<br>2. 开源方案：Mem0、Zep、LangMem | 明确Autodream在行业中的水平位置 |
| 🎯 **模块专项评估** | 五大核心模块单独测试：<br>1. 记忆整理流水线<br>2. 用户习惯识别引擎<br>3. 技能自我进化模块<br>4. 主动洞察反馈模块<br>5. 基础性能指标 | 量化每个模块的准确率和效果 |
| 🚀 **端到端全流程评估** | 模拟真实用户全周期使用场景，测试完整Autodream能力的用户体验提升 | 验证整体价值 |

## 三、测试数据集方案
### 3.1 三类数据集构成
| 数据集类型 | 内容 | 规模 | 用途 |
|------------|------|------|------|
| 🌐 **公开基准数据集** | 行业通用标准：<br>1. LoCoMo（通用记忆能力）<br>2. LongMemEval（长时记忆召回）<br>3. BEAM（大规模性能） | 各10K+样本 | 和行业效果对齐，横向对比 |
| 📝 **Autodream专项测试集** | 针对我们的功能定制：<br>1. 记忆分类测试集（5类记忆各100条标注样本）<br>2. 习惯识别测试集（4类习惯各100条标注样本，包含20条冲突样本）<br>3. 技能生成测试集（50条成功任务轨迹，包含重复任务、复杂任务）<br>4. 主动洞察测试集（30个真实用户场景） | 总500条标注样本 | 专项测试各模块准确率 |
| 🧪 **消融实验数据集** | 定制对比测试集：<br>1. 重复记忆测试集（50对重复/相似记忆）<br>2. 技能重复测试集（30组功能相似描述不同的技能）<br>3. 时序习惯测试集（20条周期性习惯样本） | 总100条样本 | 测试单个模块的贡献，做消融实验 |

### 3.2 数据集存储位置
```
tests/evaluation/
├── datasets/
│   ├── public/          # 公开基准数据集
│   ├── autodream/       # 专项测试集
│   └── ablation/        # 消融实验数据集
└── results/             # 测试结果输出目录
```

## 四、各模块评估指标与方法
### 4.1 记忆整理流水线评估
| 指标 | 定义 | 合格阈值 | 测试方法 |
|------|------|----------|----------|
| 记忆分类准确率 | 正确分类到5大类的记忆占比 | ≥90% | 用标注好的记忆分类测试集测试，计算正确分类的比例 |
| 去重准确率 | 正确识别并合并重复记忆的比例 | ≥95% | 用重复记忆测试集测试，计算正确合并的重复对数比例 |
| 冲突检测准确率 | 正确识别矛盾记忆的比例 | ≥85% | 用包含冲突样本的习惯测试集测试，计算正确识别的冲突比例 |
| 压缩信息保留率 | 压缩后的摘要包含核心信息的比例 | ≥80% | LLM Judge评估压缩前后的信息保留度 |
| 归档准确率 | 正确归档过期会话的比例 | 100% | 模拟生成不同时间的会话，验证归档逻辑正确性 |

### 4.2 用户习惯识别引擎评估
| 指标 | 定义 | 合格阈值 | 测试方法 |
|------|------|----------|----------|
| 习惯识别准确率 | 正确识别习惯的比例 | ≥85% | 用标注好的习惯测试集测试，计算正确识别的比例 |
| 冲突检测准确率 | 正确识别矛盾习惯的比例 | ≥90% | 用包含冲突样本的测试集测试，计算正确识别的冲突比例 |
| 时序模式识别准确率 | 正确识别周期性习惯的比例 | ≥80% | 用时序习惯测试集测试，计算正确识别的周期性习惯比例 |
| 置信度校准度 | 置信度和实际准确率的相关性 | ≥0.8 | 统计不同置信度区间的实际准确率，计算皮尔逊相关系数 |

### 4.3 技能自我进化模块评估
| 指标 | 定义 | 合格阈值 | 测试方法 |
|------|------|----------|----------|
| 技能生成准确率 | 生成可用技能的比例 | ≥80% | 用技能生成测试集测试，LLM Judge评估生成的技能是否可用 |
| 重复技能识别准确率 | 正确识别相似技能的比例 | ≥95% | 用技能重复测试集测试，计算正确识别的重复技能组比例 |
| 技能优化成功率 | 优化后技能成功率提升幅度 | ≥10% | 模拟低成功率技能，验证优化后的提升效果 |
| agentskills兼容性 | 导入/导出的技能可正常使用比例 | 100% | 测试导出再导入的技能是否功能一致 |

### 4.4 主动洞察反馈模块评估
| 指标 | 定义 | 合格阈值 | 测试方法 |
|------|------|----------|----------|
| 提醒准确率 | 正确的周期性提醒比例 | ≥90% | 用专项测试集测试，验证提醒的正确性和时机合理性 |
| 建议相关性 | 优化建议和用户习惯的匹配度 | ≥85% | LLM Judge评估建议的相关性和实用性 |
| 风险预警准确率 | 正确的风险预警比例 | 100% | 风险预警不能有误报，漏报率≤5% |
| 技能推荐准确率 | 推荐技能和当前场景的匹配度 | ≥80% | 测试推荐技能是否符合当前任务需求 |

### 4.5 基础性能指标
| 指标 | 定义 | 合格阈值 | 测试方法 |
|------|------|----------|----------|
| 单次执行耗时 | Autodream单次完整运行时间 | ≤30s（1万条记忆规模） | 压测不同记忆规模下的执行时间 |
| 内存占用 | 运行时内存占用 | ≤512MB | 监控运行时内存峰值 |
| 数据库大小增长 | 每月数据增长 | ≤100MB/用户 | 统计每月新增的记忆、习惯、技能存储大小 |
| 无干扰性 | Autodream运行对主流程的影响 | 0影响 | 验证Autodream后台运行不阻塞主交互流程 |

### 5.4 运行方式
#### 1. 安装依赖
```bash
pip install -r tests/evaluation/requirements.txt
```

#### 2. 配置环境变量
```env
# 可选，用于LLM Judge自动评估
LLM_BASE_URL=xxx
LLM_API_KEY=xxx
LLM_MODEL=qwen3.5-35b
```

#### 3. 执行全量评估
```bash
# 运行全部评估
python tests/evaluation/run_evaluation.py --all

# 只运行指定模块评估
python tests/evaluation/run_evaluation.py --module memory,habit

# 运行横向对比评估
python tests/evaluation/run_evaluation.py --comparison
```

#### 4. 生成评估报告
```bash
python tests/evaluation/generate_report.py --input results/latest.json --output report.md
```
