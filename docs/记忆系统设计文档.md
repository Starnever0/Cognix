## 一、概述

Cognix 记忆系统是为 OpenClaw Agent 系统设计的外挂记忆模块，提供完整的短期/长期记忆管理能力，支持记忆的增删改查、关联推理、置信度衰减等核心功能。

### 核心价值

| 特性 | 说明 |
|------|------|
| **双轨存储** | SQLite + Markdown 文件，兼顾结构化查询和人类可读 |
| **分层记忆** | 短期记忆（会话级别）+ 长期记忆（持久存储） |
| **三类记忆** | 事实记忆、偏好记忆、经验记忆 |
| **Agent友好** | 提供标准接口，支持上下文注入 |
| **可解释性** | 记忆以MD文件存储，用户可直接查看和编辑 |

## 三、核心模块设计

### 3.1 短期记忆 (Short-term Memory)

**存储位置**：SQLite 内存表

**生命周期**：会话绑定，默认 24 小时过期

**用途**：存储当前会话的临时上下文信息

#### 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| session_id | TEXT | 会话ID |
| key | TEXT | 记忆键 |
| value | TEXT | 记忆值（JSON序列化） |
| confidence | REAL | 置信度 |
| created_at | TIMESTAMP | 创建时间 |
| expires_at | TIMESTAMP | 过期时间 |

---

### 3.2 长期记忆 (Long-term Memory)

**存储位置**：SQLite + Markdown 文件双轨存储

**生命周期**：永久存储，带置信度衰减机制

**三类记忆类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| fact | 事实记忆 | 用户身份、联系人信息、固定配置 |
| preference | 偏好记忆 | 输出格式、使用习惯、时间偏好 |
| experience | 经验记忆 | 任务执行经验、工具调用策略 |

#### 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| memory_type | TEXT | 记忆类型 |
| key | TEXT | 记忆键 |
| value | TEXT | 记忆值（JSON序列化） |
| confidence | REAL | 置信度（0-1） |
| md_file_path | TEXT | MD文件路径 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| accessed_at | TIMESTAMP | 访问时间 |

---

### 3.3 记忆关联 (Memory Links)

**用途**：建立记忆之间的语义关联，支持推理

**关联类型**：

| 类型 | 说明 |
|------|------|
| related_to | 相关关系 |
| derived_from | 派生关系 |
| contradicts | 矛盾关系 |

## 四、Agent 接口设计

### 4.1 接口类：AgentMemoryClient

#### remember(key, value, memory_type="fact", confidence=1.0)

**功能**：添加长期记忆

**参数**：
- key: str - 记忆键
- value: Any - 记忆值
- memory_type: str - 记忆类型（fact/preference/experience）
- confidence: float - 置信度（0-1）

**返回**：MemoryQueryResponse

#### recall(key=None, query=None)

**功能**：检索记忆

**参数**：
- key: str - 记忆键（精确匹配）
- query: str - 搜索关键词（模糊搜索）

**返回**：MemoryQueryResponse

#### forget(key)

**功能**：删除记忆

**参数**：
- key: str - 记忆键

**返回**：MemoryQueryResponse

#### contextualize(session_id, query=None)

**功能**：为Agent准备上下文记忆

**参数**：
- session_id: str - 会话ID
- query: str - 查询文本（用于搜索相关记忆）

**返回**：MemoryQueryResponse

#### learn(session_id, user_input, agent_response, tool_calls=None)

**功能**：记录Agent交互到短期记忆

**参数**：
- session_id: str - 会话ID
- user_input: str - 用户输入
- agent_response: str - Agent响应
- tool_calls: List - 工具调用记录

**返回**：MemoryQueryResponse

#### connect(key1, key2, relation="related_to")

**功能**：建立记忆关联

**参数**：
- key1: str - 源记忆键
- key2: str - 目标记忆键
- relation: str - 关联类型

**返回**：MemoryQueryResponse

#### get_connections(key, relation_type=None)

**功能**：获取记忆关联

**参数**：
- key: str - 记忆键
- relation_type: str - 关联类型过滤

**返回**：MemoryQueryResponse

---

### 4.2 返回结构：MemoryQueryResponse

```python
{
    "success": bool,    # 操作是否成功
    "data": Any,        # 返回数据
    "message": str      # 状态消息
}
```

## 六、核心算法

### 6.1 置信度衰减机制

```
if memory.accessed_at < (now - days_threshold):
    memory.confidence = max(0.1, memory.confidence - decay_rate)
```

### 6.2 记忆搜索算法

```
1. 关键词匹配 key 和 value 字段
2. 按 confidence 降序排序
3. 按 accessed_at 降序排序（最近访问优先）
4. 返回前 N 条结果
```

## 七、与 OpenClaw 的集成方式

### 7.1 推荐集成模式

```python
from cognix.interfaces.agent import AgentMemoryClient

class OpenClawAgent:
    def __init__(self):
        self.memory = AgentMemoryClient()
    
    def think(self, session_id, user_input):
        # 获取上下文记忆
        context = self.memory.contextualize(session_id, user_input)
        
        # 将记忆注入prompt
        prompt = f"""
        记忆信息：{context.data}
        
        用户输入：{user_input}
        """
        
        # 调用LLM推理
        response = self.llm.generate(prompt)
        
        # 记录交互到记忆系统
        self.memory.learn(session_id, user_input, response)
        
        return response
```

### 7.2 工作流程

```
用户输入 → Agent → contextualize() → 获取记忆 → 注入Prompt → LLM推理 → learn() → 记录交互 → 返回响应
```

## 九、安全性考虑

1. **权限控制**：每个记忆可关联用户ID，限制跨用户访问
2. **数据加密**：敏感记忆可加密存储
3. **审计日志**：记录所有记忆操作，便于追溯
4. **过期清理**：定期清理过期短期记忆
