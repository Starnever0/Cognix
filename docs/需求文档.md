## 一、项目背景
在企业办公场景中，用户普遍存在稳定且重复的工作习惯，例如：
- 每周五下午整理周报
- 开会前 5 分钟准备会议资料
- 输出内容偏好表格格式
- 周报默认发送给直属负责人
- 高频使用固定命令或固定流程

当前大模型普遍缺乏持续个性化记忆能力，每次对话都需要用户重新说明偏好，无法长期观察用户行为并主动提供服务。

因此，我们计划构建 **Cognix**：面向企业 AI Agent 的个人工作习惯与偏好记忆引擎。
> Cognix learns user habits, remembers preferences, and proactively assists workplace workflows.

## 二、项目定位与核心目标
### 2.1 项目定位
Cognix 是一套面向个人飞书办公场景的 Memory Engine，通过观察用户行为信号，自动学习工作习惯、使用偏好与周期规律，并在合适时机主动提供智能协助。

系统重点解决以下痛点：
1. 用户偏好需要反复说明，缺乏持续记忆
2. 重复性任务依赖手动触发，效率低下
3. Agent 无法理解个人工作节奏
4. 个性化体验弱，无法主动服务

### 2.2 核心目标
构建完整的工作流闭环：
**行为观察 → 习惯学习 → 偏好记忆 → 规则生成 → 主动服务**

实现以下 5 大核心能力：
| 能力 | 说明 |
|------|------|
| Remember | 记住用户长期偏好 |
| Learn | 从行为中发现规律 |
| Trigger | 在合适时机主动提醒或执行 |
| Adapt | 根据新行为动态调整规则 |
| Personalize | 提供个性化办公体验 |

## 三、总体架构设计
```
User / Feishu / CLI / Bot
        ↓
Behavior Event Collector（行为事件收集器）
        ↓
Cognix Memory Engine
-------------------------------------------------
Preference Store | Pattern Miner | Rule Engine
Retriever | Updater | Trigger Scheduler
-------------------------------------------------
        ↓
Proactive Assistant / OpenClaw Agent（主动助手/智能体）
```

### 3.1 数据来源（可落地场景）
不依赖全量点击流，仅采集高价值行为事件：
| 数据类型 | 具体内容 |
|----------|----------|
| 飞书 Bot 交互事件 | @机器人请求周报、选择输出格式、点击卡片按钮、请求提醒服务 |
| CLI 行为日志 | 高频命令调用、固定参数组合、周期性使用行为 |
| 日程/Calendar 事件 | 周期会议、截止时间、周报时间点 |
| 文档行为（可选接入） | 周期性创建周报文档、高频访问项目文档 |
| 显式反馈数据 | 用户确认/拒绝建议、修改输出格式 |

## 四、核心模块设计
### 4.1 Preference Store（偏好记忆模块）
存储用户长期偏好数据，示例结构：
```json
{
  "report_format": "table",
  "weekly_report_receiver": "Leader A",
  "meeting_reminder": "5min_before"
}
```

### 4.2 Pattern Miner（规律发现模块）
从历史行为中挖掘周期规律，示例输出：
> 连续 4 周周五 16:00 请求生成周报，置信度：0.92

### 4.3 Rule Engine（规则生成模块）
将挖掘到的规律转化为可执行规则，示例结构：
```json
{
  "trigger": "Friday 15:55",
  "action": "generate_weekly_report"
}
```

### 4.4 Trigger Scheduler（主动服务模块）
负责触发提醒或自动执行任务，支持场景：
- 周报草稿生成
- 会前材料提醒
- 高频命令推荐
- 输出格式自动适配

## 五、测试场景与用例
### 场景 A：周报助手（主场景）
**前置条件：** 过去 4 周用户均在周五下午请求写周报
**预期结果：** 第 5 周系统主动提醒：「今天是周五，已按你偏好的表格格式生成本周周报草稿，是否发送给 Leader A？」

### 场景 B：会议助手
**前置条件：** 检测到用户每周一 10:00 开例会
**预期结果：** 系统在 9:55 主动提醒：「距离会议还有 5 分钟，已整理上周待办事项。」

### 场景 C：输出偏好学习
**前置条件：** 用户连续多次选择「表格输出」
**预期结果：** 后续请求默认采用表格格式，无需用户重复说明

### 场景 D：CLI 工作流推荐
**前置条件：** 用户常使用命令 `report summary --weekly --table`
**预期结果：** 用户输入命令前缀时自动补全推荐

## 七、风险与挑战
1. **行为数据有限：** 比赛环境难以获得真实大规模数据，需要通过事件日志 + 模拟数据补足
2. **规律识别误报：** 偶发行为不能被误识别为长期习惯，需要设置频次阈值与置信度机制
3. **主动提醒打扰用户：** 触发频率需控制，避免过度打扰
4. **偏好变化：** 用户习惯会变化，需要支持动态更新与衰减机制
5. **隐私与权限：** 仅使用授权范围内的行为数据

## 九、项目愿景
Cognix 要做的不是被动记住用户说过什么，而是**主动理解用户如何工作，并在正确的时间提供正确的帮助**。
