# Cognix记忆系统设计文档

## 设计理念

本记忆系统的设计参考了OpenClaw和ReMe等主流项目，遵循以下核心原则：

1. **Markdown作为真相来源**：所有记忆以人类可读的Markdown格式存储
2. **SQLite作为索引缓存**：SQLite用于快速检索，可随时从Markdown重建
3. **短期/长期记忆分离**：
   - 短期记忆：会话级别，内存存储
   - 长期记忆：持久化，Markdown文件存储

## 文件存储结构

### 目录结构

```
{config.home_path}/
├── memory/
│   ├── 2026-04-25.md      # 每日记忆日志
│   ├── 2026-04-26.md
│   └── ...
├── MEMORY.md              # 持久长期记忆
└── cognix.db              # SQLite索引（可重建）
```

### Markdown文件格式

#### 每日日志 (memory/YYYY-MM-DD.md)

```markdown
# 2026-04-25 日志

## 事实记忆

张三，28岁，软件工程师

## 用户偏好

喜欢简洁回答，偏好深色主题

## 项目笔记

正在开发Cognix记忆系统
```

#### 持久记忆 (MEMORY.md)

```markdown
# 持久记忆

## 基本信息

姓名：张三
职业：软件工程师

## 常用命令

- git status
- git commit
- git push
```

## 核心功能

### 1. 添加记忆

**方法签名：**

```python
def add_memory(self, heading: str, content: str, date: Optional[datetime] = None)
```

- 添加到每日日志文件
- 自动更新索引
- 按##标题组织内容

**示例：**

```python
memory.add_memory("用户偏好", "喜欢简洁回答")
```

### 2. 添加持久记忆

**方法签名：**

```python
def add_persistent_memory(self, heading: str, content: str)
```

- 添加到MEMORY.md
- 跨会话持久化

### 3. 搜索记忆

**方法签名：**

```python
def search_memory(self, query: str, limit: int = 10, 
                 source: Optional[str] = None) -> List[Dict]
```

搜索优先级：
1. SQLite索引（chunks表）
2. 直接读取Markdown文件（回退）

**返回格式：**

```python
{
    'id': 'chunk-uuid',
    'path': '/path/to/file.md',
    'source': 'memory|persistent',
    'start_line': 4,
    'end_line': 7,
    'text': '## 标题\n\n内容...',
    'score': 1.0
}
```

### 4. 获取上下文

**方法签名：**

```python
def get_daily_context(self, days_back: int = 1) -> str
```

- 获取最近N天的记忆
- 合并短期记忆
- 作为Agent的上下文

### 5. 短期记忆

```python
# 添加
memory.add_short_term(session_id, key, value)

# 获取
memory.get_short_term(session_id)

# 清除
memory.clear_short_term(session_id)
```

### 6. 重建索引

```python
memory.rebuild_index()
```

从Markdown文件完全重建SQLite索引。

## Agent接口

### 快速开始

```python
from cognix.interfaces.agent import AgentMemoryClient

# 初始化
client = AgentMemoryClient()

# 记住内容
client.remember("用户信息", "姓名：张三，28岁")

# 搜索
results = client.recall("张三")

# 获取上下文
context = client.contextualize(session_id)

# 短期记忆
client.note_short_term(session_id, "task", "完成文档")
```

### API方法

| 方法 | 说明 |
|------|------|
| `remember(heading, content, persistent=False)` | 添加记忆 |
| `recall(query, source=None, limit=10)` | 搜索记忆 |
| `contextualize(session_id=None, days_back=1)` | 获取上下文 |
| `note_short_term(session_id, key, value)` | 添加短期记忆 |
| `get_short_term(session_id)` | 获取短期记忆 |
| `clear_session(session_id)` | 清除会话记忆 |
| `rebuild_index()` | 重建索引 |

## 优势

1. **人类可读**：Markdown文件可直接编辑
2. **可重建**：索引随时可从Markdown重建
3. **按时间组织**：每日日志便于追溯
4. **分离关注点**：持久/临时，长期/短期清晰分离
5. **主流兼容**：遵循OpenClaw/ReMe实践
